## 一、日常操作

### 启动 / 重启

```bash
# 重启容器（自动启动全部 8 个节点）
ssh pi@192.168.31.75
docker restart patrol_car

# 验证
docker exec patrol_car head -2 /tmp/cmd_vel_bridge.log
# 应看到: Rosmaster 就绪 (port=/dev/myserial)
```

### Web 面板

```
http://192.168.31.75:5000
```
- D-pad 遥控（按住持续移动，松开停止）
- 实时画面 + SLAM 地图
- 系统监控（电池/CPU/内存/温度/磁盘/负载）
- 抓拍管理、路线编辑、报警记录

### 语音控制

| 语音命令 | 动作 |
|:---------|:-----|
| 前进 | vx=0.2 m/s 持续 2s |
| 后退 | vx=-0.2 m/s 持续 2s |
| 左转 | wz=0.8 rad/s 持续 2s |
| 右转 | wz=-0.8 rad/s 持续 2s |
| 停止 | 零速 |

## 二、自主探索建图

### 启动建图

```bash
docker exec -it patrol_car bash /home/pi/patrol_robot/scripts/start_slam.sh
```

启动链: 静态 TF → slam_toolbox → explore_runner → 自动探索

3 秒后自动开始，地图实时在 Web :5000 显示。

### 停止建图 + 停车

```bash
docker exec patrol_car bash /home/pi/patrol_robot/scripts/stop_slam.sh
```

### 查看日志

```bash
# 探索日志（状态/前沿/避障）
docker exec patrol_car tail -f /tmp/explore.log

# SLAM 日志
docker exec patrol_car tail -f /tmp/slam.log

# 底盘桥日志（速度/编码器/odom）
docker exec patrol_car tail -f /tmp/cmd_vel_bridge.log
```

### 保存地图

停止脚本不会自动保存。手动保存：

```bash
docker exec patrol_car bash -c '
  source /opt/ros/foxy/setup.bash &&
  source /home/pi/patrol_robot/patrol_robot/install/setup.bash &&
  ros2 run nav2_map_server map_saver_cli -f /home/pi/patrol_robot/maps/my_map
'
# 生成 my_map.pgm + my_map.yaml
```

## 三、手动遥控（命令行）

### 用 ros2 topic pub

```bash
# 进入容器环境
docker exec -it patrol_car bash
source /opt/ros/foxy/setup.bash
source /home/pi/patrol_robot/patrol_robot/install/setup.bash

# 前进 0.15 m/s（持续发 2 秒）
for i in $(seq 1 20); do
  ros2 topic pub -1 /cmd_vel geometry_msgs/msg/Twist '{linear: {x: 0.15}}'
  sleep 0.1
done
# 停止
ros2 topic pub -1 /cmd_vel geometry_msgs/msg/Twist '{}'

# 后退
ros2 topic pub -1 /cmd_vel geometry_msgs/msg/Twist '{linear: {x: -0.15}}'

# 左转（角速度正值=逆时针）
ros2 topic pub -1 /cmd_vel geometry_msgs/msg/Twist '{angular: {z: 0.5}}'

# 右转
ros2 topic pub -1 /cmd_vel geometry_msgs/msg/Twist '{angular: {z: -0.5}}'

# 停车
ros2 topic pub -1 /cmd_vel geometry_msgs/msg/Twist '{}'
```

### 用 Web API

```bash
# 前进 0.5 秒，速度 0.15
curl -X POST http://192.168.31.75:5000/api/control/move \
  -H 'Content-Type: application/json' \
  -d '{"direction":"forward","duration":0.5,"speed":0.15}'

# 左转
curl -X POST http://192.168.31.75:5000/api/control/move \
  -H 'Content-Type: application/json' \
  -d '{"direction":"left","duration":1.0,"speed":0.15}'

# 停止
curl -X POST http://192.168.31.75:5000/api/control/move \
  -H 'Content-Type: application/json' \
  -d '{"direction":"stop"}'

# 调速
curl -X POST http://192.168.31.75:5000/api/control/speed \
  -H 'Content-Type: application/json' \
  -d '{"speed": 0.2}'
```

| direction | 说明 | 速度 |
|:----------|:-----|:-----|
| forward | 前进 | vx = speed |
| backward | 后退 | vx = -speed |
| left | 左转 | wz = speed×5 |
| right | 右转 | wz = -speed×5 |
| strafe_left | 左移 | vy = speed |
| strafe_right | 右移 | vy = -speed |
| stop | 停止 | 零速 |

### 急停

```bash

  -H 'Content-Type: application/json' -d '{"direction":"stop"}'

# 方法 3: 停止脚本
docker exec patrol_car bash /home/pi/patrol_robot/scripts/stop_slam.sh
```

## 五、故障排查

### 车不动

```bash
# 1. 看 bridge 日志
docker exec patrol_car tail -10 /tmp/cmd_vel_bridge.log

# recv=0  → 发布端没在发（检查 Web/语音/explore）
# recv>0 但 vx=0.00  → 发的是零速（正常等待）
# vx=0.15 但 enc 不变  → STM32 通信断了（见下）
```

### STM32 通信断（enc 全 0）

```bash
# 确认
docker exec patrol_car tail -3 /tmp/cmd_vel_bridge.log
# 如果 enc=(0, 0, 0, 0) 且有 Exception → STM32 死锁

# ⚠️ 绝不跑串口探测脚本！
# 解决: 车体物理断电 → 等 5 秒 → 重新通电 → docker restart patrol_car
```

### USB 串口漂移

```bash
# 检查符号链接
ls -la /dev/myserial /dev/myspeech /dev/rplidar

# 如果缺失，重载 udev
sudo udevadm control --reload-rules
sudo udevadm trigger --subsystem-match=tty

# udev 规则文件
cat /etc/udev/rules.d/99-yahboomcar.rules
```

### 容器反复重启

```bash
docker logs patrol_car 2>&1 | tail -30
docker restart patrol_car
```

### 摄像头被抢占

```bash
# 释放 /dev/video0
docker exec patrol_car fuser -k /dev/video0
docker restart patrol_car
```

## 六、速度参考（含运动补偿）

> bridge 的 compensate_motion() 会放大命令值，下表为**用户期望速度**（发到 /cmd_vel 的值）

| 场景 | vx (m/s) | wz (rad/s) | 说明 |
|:-----|:---------|:-----------|:-----|
| 慢速前进 | 0.10 | 0 | 探索建图 |
| 正常前进 | 0.15 | 0 | Web 遥控默认 |
| 快速前进 | 0.20 | 0 | 语音控制 |
| 缓慢转向 | 0 | 0.3 | 探索找前沿 |
| 正常转向 | 0 | 0.5 | 手动遥控 |
| 快速转向 | 0 | 0.8 | 语音控制 |
| 死区 | <0.02 | <0.02 | bridge 自动归零 |

### 运动补偿系数

| 分量 | 补偿公式 | 说明 |
|:-----|:---------|:-----|
| vx | vx × 1.05 | 接近线性，略微增益 |
| vy | 1.5 × vy + 0.15 | 侧移阻力大（前固定滚轮） |
| wz | 2.33 × wz + 0.4 | 死区 + 增益（前固定滚轮刮地） |

## 八、注意事项

1. **绝不跑串口探测脚本**：逐个打开 ttyUSB 测 Rosmaster 会导致 STM32 死锁，只能断电恢复
2. **STM32 死锁表现**：bridge 日志 enc=(0,0,0,0) + Exception → 断电重启
3. **重启 3 次验证**：每次修改后重启 3 次确认稳定
4. **充电时**：车架空轮子悬空可转，但 odom 不走距离，建图找不到前沿（正常）
5. **Nav2 未运行**：巡逻状态机检测到 Nav2 不可用会降级为 IDLE，只有 Web 遥控和 YOLO 检测可用
6. **运动补偿**：所有 /cmd_vel 速度会被 bridge 放大，wz 实际 = 2.33×命令 + 0.4
