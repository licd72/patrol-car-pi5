# 调试黄金法则

## 法则一：设备路径问题 → 先查 udev，不要改代码

树莓派重启后 USB 枚举顺序会变。CH340 芯片没有唯一序列号，只能靠 USB 物理路径区分。

### 正确排障流程
1.  — 检查符号链接
2.  — 查看当前路径
3. 更新  中的 ID_PATH
4. 

### 错误做法
- 改程序里的串口路径
- 加重试逻辑  
- 改架构绕过问题

## 法则二：先最小化测试，再改架构
1. 硬件层: 串口能打开吗
2. 驱动层: 单独跑驱动有无报错（不要 2>/dev/null！）
3. 通信层: 话题有发布者/订阅者吗
4. 最后才改架构

## 法则四：STM32 固件不会自动停止
## 法则五：环境变量在容器中不会自动继承
